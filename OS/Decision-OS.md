# Decision-OS：决策操作系统

## 概述

Decision-OS 是一个系统性决策框架。它的目标是帮助你在面对重要决策时，不依赖直觉和情绪，而是遵循一套可靠的流程。

## 核心原则

1. **决策质量 ≠ 结果质量**：好决策可能得到坏结果，坏决策可能得到好结果。评估决策，而非结果。
2. **写下来**：任何重要决策都要书面化。书写强迫你理清思路。
3. **冷却期**：重大决策至少 24 小时的冷却期。
4. **魔鬼代言人**：强制寻找反对自己观点的最强理由。

## 决策流程

### 第一阶段：定义问题

```
- [ ] 我真正在决定什么？
- [ ] 这是一个可逆决策还是不可逆决策？
- [ ] 这个决策的时间窗口有多长？
- [ ] 不决策的代价是什么？
```

### 第二阶段：信息收集

```
- [ ] 我需要哪些关键信息？
- [ ] 我的信息来源可靠吗？
- [ ] 我是否有确认偏误？（只找支持自己观点的信息）
- [ ] 我是否考虑了"未知的未知"？
```

### 第三阶段：选项生成

```
- [ ] 列出所有可能的选项（包括"什么都不做"）
- [ ] 有没有我没想到的选项？
- [ ] 如果我是我的对手/竞争者，我会怎么做？
```

### 第四阶段：评估与选择

```
- [ ] 每个选项的期望值是多少？（概率 × 收益/损失）
- [ ] 每个选项最坏情况是什么？我能承受吗？
- [ ] 10 分钟后看这个决策：我会后悔吗？
- [ ] 10 年后看这个决策：它还重要吗？
```

### 第五阶段：执行与回顾

```
- [ ] 写下决策理由和预期结果
- [ ] 设定回顾时间点
- [ ] 实际结果 vs 预期结果：差距在哪？学到了什么？
```

## 重要决策类型

### 投资决策

参考 `Financial-OS.md`，但加入决策层过滤：

- 这是投资还是投机？（诚实回答）
- 我比市场多知道什么？（信息优势）
- 如果这个行业明天开始衰退，我还会持有吗？

### 职业决策

- 这份工作/项目是在建立复利，还是只是用时间换钱？
- 5 年后回头看，这个选择会让我更接近目标还是更远？
- 我在跟谁一起工作？他们是我想要成为的人吗？

### 生活决策

- 这个决定会让我 80 岁时感到骄傲还是遗憾？
- 我是因为恐惧还是因为渴望而做这个决定？
- 如果金钱不是问题，我会怎么选？（然后找出实现这个选择的途径）

## 认知偏误防护清单

每次重大决策前必须过一遍：

```
- [ ] 确认偏误：我是否在寻找反面证据？
- [ ] 锚定效应：我是否被某个数字/信息"锚定"了？
- [ ] 损失厌恶：我是否因为害怕损失而拒绝有正期望值的机会？
- [ ] 羊群效应：我是否因为"别人都这么做"而跟随？
- [ ] 过度自信：我是否高估了自己的判断力？
- [ ] 近因偏误：最近的事件是否过度影响了我的判断？
```

## 与其他 OS 的关系

- **Financial-OS**：当决策涉及资金配置时联动
- **Business-OS**：当决策涉及商业判断时联动
- **Creator-OS**：当决策涉及创造性项目时联动

Decision-OS 是所有 OS 的"中枢"——每个领域的具体决策最终都通过 Decision-OS 来处理。
